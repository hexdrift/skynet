# Skynet — Design Context

DSPy-as-a-service: run, compare, and serve optimized LLM programs without touching the DSPy internals yourself. Hebrew-first UI, Vercel-inspired aesthetic, warm monochromatic T-800 palette.

## Design Context

### Users
External developers who build with LLMs but are **not** data scientists or ML specialists. They know how to write code and call an API, but they don't want to reason about optimizers, prompt-tuning strategies, or evaluation harnesses. Their job: "I have a task and a dataset — give me a program that performs well, fast, without making me learn DSPy." The interface must let a competent backend engineer ship an optimization without reading a paper.

### Brand Personality
**Easy · Reliable · Valuable** — three words, in that order.

- **Voice**: clear, factual, no jargon. Speak to a smart engineer who hasn't read the ML literature. Explain the *why* in one sentence, never three.
- **Tone**: calm and confident. Never hype, never cute, never apologetic.
- **Emotional goal**: the user should feel *in control* and *unhurried*. Submitting an optimization should feel like sending a well-formed API request — predictable, boring in the good way. No anxiety, no FOMO, no performative polish.

### Aesthetic Direction
**Reference: Vercel.** Restrained typography, generous whitespace, decisive contrast, tiny delightful motion details, zero ornament. Content-first — chrome only exists when it earns its keep.

- **Palette**: warm monochromatic — cream background (`#faf8f5`), deep brown foreground (`#1c1612`), dark-brown primary (`#3d2e22`). Chart ramp is a 5-step brown gradient. Accent colors (`--danger`, `--success`, `--warning`) are desaturated and used sparingly for state, never decoration.
- **Theme**: light mode only (dark tokens exist but are force-disabled in `theme-provider.tsx`). Don't design for dark unless explicitly asked.
- **Typography**: **Heebo Variable** for body/UI (Hebrew + Latin), **Inter Variable** for headings. Base 16px. The current type scale is under-expressed — headings and body feel too close in weight/size. There's room to strengthen the hierarchy.
- **Motion**: fast and snappy. `--duration-fast: 120ms`, `--duration-base: 160ms`, `--ease-snappy: cubic-bezier(0.2, 0.8, 0.2, 1)`. Framer Motion is available; use it for state transitions, not decoration.
- **Radius**: `0.5rem` base. Cards and inputs should feel softly cut, not pill-shaped.
- **Iconography**: Lucide only, stroke 1.5–2, matched to text color.

### Design Principles
1. **Boring is a feature.** A user who never notices the UI had a successful session. Prefer the dull, correct affordance over the clever one.
2. **Hebrew is the primary language, not an afterthought.** Everything must read naturally RTL: flow, punctuation, line-height, numeric alignment, icon direction. Test in Hebrew first, Latin second. Mixed-direction strings (English code inside Hebrew sentences) must not break.
3. **Type hierarchy carries the page.** Don't lean on color, borders, or cards to build structure — use size, weight, and whitespace. The warm palette is intentionally narrow so typography has to do the work.
4. **Whitespace over decoration.** When in doubt, remove. No gradients, no shadows unless elevation is real, no emoji, no badges that don't denote state.
5. **Explain the ML, don't expose it.** The user is not a data scientist. Labels, help text, and defaults should translate DSPy concepts into plain engineering language. If a setting can't be explained in one sentence, it probably shouldn't be a user-facing setting.

### Accessibility
- **WCAG AA baseline.** Contrast, focus rings, keyboard reachability — non-negotiable on every new component.
- **Hebrew typography is a hard constraint.** Heebo is chosen for readability; don't swap it without checking. Watch line-height (Hebrew descenders sit differently than Latin), avoid italic (Hebrew has no true italic), and never letter-space Hebrew text.
- **Mixed-direction content**: English identifiers (file names, class names, code) inside Hebrew UI must render correctly — use `dir="ltr"` or `bdi` where needed.
- **Reduced motion**: respect `prefers-reduced-motion`. Motion is atmosphere, not information — nothing critical should depend on it.
